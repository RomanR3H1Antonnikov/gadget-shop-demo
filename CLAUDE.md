# CLAUDE.md — Telegram-бот для магазина электроники (демо IT-студии)

## Контекст проекта

Этот бот — **демо-продукт IT-студии** для продажи ботов владельцам магазинов электроники (Горбушка, SMB-ритейл).
Цель демо: владелец магазина должен сказать *«хочу такой же себе»*.

Демо-магазин: **GadgetPro** — техника Apple, Samsung, Xiaomi, аксессуары, Trade-in.

**Принципы кода:**
- Никакого хардкода — всё через конфиг и БД
- Чистый код с комментариями на русском
- Легко адаптируется под любой магазин (замени `.env` и перезапусти `seed.py`)
- Каждый модуль независим и переиспользуем

---

## Этап 1: Каталог — `handlers/catalog.py`

### Логика навигации:

```
/start или кнопка "Каталог 📦"
    → показать корневые категории (у кого parent_id IS NULL)
    → нажал категорию → если есть подкатегории, показать их
    → нажал подкатегорию → показать товары (фото + цена + кнопка "Подробнее")
    → нажал "Подробнее" → карточка товара: фото, название, описание, цена, кнопки ["Заказать 🛒", "Назад"]
    → кнопка "Назад" → возврат на предыдущий уровень
```

### Главное меню (Reply-клавиатура):
```
[Каталог 📦]  [Подобрать товар 🤖]
[Мои заказы 📋]  [О магазине ℹ️]
```

### Callback data format:
- `cat_{id}` — открыть категорию
- `product_{id}` — открыть карточку товара
- `order_{id}` — начать заказ товара
- `back_to_cat_{id}` — вернуться в категорию
- `back_to_main` — главное меню

### Карточка товара:
- Отправляй `send_photo` с `caption` (если `photo_url` есть)
- Если фото нет — `send_message` с текстом
- Формат: `**Название**\n\nОписание\n\n💰 Цена: X ₽\n✅ В наличии`

## Этап 2: Заказы — `handlers/orders.py`

### FSM States (`OrderStates`):
1. `waiting_name` — "Как вас зовут?"
2. `waiting_phone` — "Введите номер телефона"  
   *(кнопка "Поделиться контактом 📱" через ReplyKeyboardMarkup)*
3. `waiting_delivery` — "Самовывоз или доставка? Укажите город или 'самовывоз'"

### После сбора данных:
1. Сохранить заказ в таблицу `orders` со статусом `new`
2. Показать пользователю подтверждение:
   ```
   ✅ Заявка #42 принята!
   
   📦 Товар: iPhone 15 Pro 256GB
   👤 Имя: Алексей
   📞 Телефон: +7 999 123-45-67
   🚚 Доставка: Москва
   
   Менеджер свяжется в течение 1 часа.
   ```
3. Отправить уведомление администратору (ADMIN_CHAT_ID):
   ```
   🛒 Новый заказ #42
   
   📦 iPhone 15 Pro 256GB (109 990 ₽)
   👤 Алексей | @username
   📞 +7 999 123-45-67
   🚚 Москва
   
   [✅ Подтвердить] [❌ Отклонить]
   ```
   Кнопки меняют статус в БД и шлют уведомление пользователю.

## Этап 3: AI-консультант — `services/openai_api.py` + `handlers/ai_consultant.py`

### Промпт-система:

```python
SYSTEM_PROMPT = """
Ты — опытный продавец-консультант магазина {shop_name}. 
Твоя задача — помочь покупателю выбрать идеальный товар.

Текущий каталог магазина:
{catalog_json}

Правила:
1. Рекомендуй ТОЛЬКО товары из каталога выше
2. Будь конкретным: называй модель и цену
3. Объясняй, почему именно этот товар подойдёт для задачи покупателя
4. Если бюджет не указан — спроси
5. Отвечай дружелюбно, без канцелярита
6. Максимум 3 рекомендации за раз
7. В конце предложи нажать "Заказать" под понравившимся товаром
"""
```

### Логика:
1. Пользователь нажимает "Подобрать товар 🤖"
2. Бот: "Опишите, что вам нужно. Например: *нужен телефон для работы до 80 000 ₽* или *ищу планшет для ребёнка 10 лет*"
3. Пользователь пишет запрос
4. `openai_api.py` получает актуальный каталог из БД (только `in_stock=1, is_hidden=0`), передаёт в промпт
5. Ответ GPT отображается пользователю
6. Под ответом — кнопки с рекомендованными товарами (парсить из ответа или отображать топ-3 из каталога)
7. Typing action (`bot.send_chat_action`) пока ждём ответа

## Важные детали реализации

### Обработка ошибок
- Все обращения к API и БД — в `try/except`
- Пользователь никогда не видит трейсбэк — только понятное сообщение
- Логирование через стандартный `logging`

### UX-детали (важно для демо!)
- `bot.send_chat_action(chat_id, "typing")` перед любым долгим ответом
- Editing message вместо отправки нового (где уместно) — `message.edit_text`
- Эмодзи в кнопках и сообщениях — делает бота живым
- Приветственное сообщение `/start` — красивое, с названием магазина из конфига
